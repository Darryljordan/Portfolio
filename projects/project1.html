Project Overview (September 2023 to January 2024)

The Nutriboard Platform was developed to address the critical challenge of detecting patients that may be at risk of malnutrition. This digital solution enables healthcare professionals to track, monitor, and intervene in patients' nutritional status through a secure, cloud-hosted application.

As the DevOps engineer and backend developer on this project, I was responsible for designing a resilient infrastructure on Microsoft Azure that could handle sensitive patient data while ensuring high availability and compliance with healthcare regulations.

Technical Challenges & Solutions

One of the primary challenges was creating a secure deployment pipeline that protected sensitive healthcare data while enabling rapid updates to the platform. I implemented:

  • A multi-environment architecture (development, staging, production) with appropriate isolation
  • Containerized application using Docker to ensure consistency across environments
  • Secure secret management through GitHub Secrets, preventing exposure of credentials
  • Automated CI/CD pipelines that included security scanning and compliance checks

Technologies Used

  • Cloud Infrastructure: Microsoft Azure VMs
  • Backend Development: Java Spring Boot for creating RESTful APIs that interfaced with the nutritional database
  • Containerization: Docker for packaging applications with consistent environments
  • CI/CD: GitHub Actions workflows for automated testing, building, and deployment
  • Monitoring:Grafana dashboards for real-time system health and performance metrics

Responsibilities

  • Security Implementation: Developed secure product delivery chain
  • Backend Development: Built Java microservices for data processing and integration with the frontend
  • Documentation: Created detailed technical documentation and runbooks for operations

Results & Impact

The Nutriboard Platform successfully launched with zero security incidents and 99.9% uptime. The solution is now worked on by the client in order to add new features and improve the user experience before use.

This project demonstrated my ability to implement DevOps best practices in a regulated healthcare environment while contributing to backend development in Java.
